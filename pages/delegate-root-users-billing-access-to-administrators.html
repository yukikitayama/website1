Delegate root user’s billing access to Administrators

Idea

  • By default, only the root user can see billing in the AWS console, and other users cannot access the billing data until the root user allows them to do so.
  • The root user needs to activate IAM user access to the Billing and Cost Management console and attach policies to the users or role.

Steps

  • Activate access to the Billing and Cost Management console
    • Sign in to AWS console as the root user.
    • Click account name at the top right
    • Click My Account
    • Click Edit next to IAM User and Role Access to Billing Information
    • Check Activate IAM Access
    • Click Update
  • Attach policy to grant permission to billing data
    • Sign in to AWS console as administrator
    • Go to IAM
    • Go to Policies
    • Click Create policy
    • Visual editor tab
      • Service: Choose a service, Billing
      • Actions: Specify the actions allowed in Billing, Check All Billing actions (aws-portal:*)
      • Resources: No change
      • Request conditions: No change
    • Click Next: Tags
    • Click Next: Review
    • Name
      • BillingFullAccess
    • Click Create policy
    • At Policies, search “BillingFullAccess”, and check it
    • Click Policy actions, attach
    • Check Administrators Group (or any user, role or group you want)
    • Click Attach policy

Test

  • Sign in to AWS console as a user who is attached to the new billing policy.
  • Go to AWS Cost Explorer
  • If you can see billing data, it’s successful.
