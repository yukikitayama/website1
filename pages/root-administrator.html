AWS root identity and administrator

Root identity (or root account credentials)

  • The combination of your email address and password.
  • AWS recommends you don’t use your root account credentials for everyday use.
  • Instead you should create new credentials, which means accessing AWS by a special URL and the credentials for the IAM user.

Create an administrator user for yourself

  • Make a user, Administrator.
  • Make a group, Administrators.
    • This group has an AdministratorAccess policy.

Sign in with account ID

  • Go to https://your_account_id.signin.aws.amazon.com/console/
    • Replace your_account_id with your 12 digits account id, which appears in AWS console top right after clicking your account information.
  • Username: Administrator, password: what you created.

Create alias to hide account ID

  • IAM, Dashboard, click Customize next to Sign-in URL, type the account alias

Sign in with account alias

  • Go to https://your_account_alias.signin.aws.amazon.com/console/
    • Replace your_account_alias with your newly created account alias
  • Account ID or account alias: account alias you created, Username: Administrator, password: what you created.

Reset password of a user

  • Go to Security credentials tab of the user.
  • Click manage next to Console password.
  • Set password, Custom password, type a new password
