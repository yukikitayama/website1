Depth First Search (DFS) uses stack

Idea

  • DFS is an algorithm to travel across (“traverse”) graph and tree data structures.
  • One of the major techniques to solve programming questions.
  • There are two implementations; using a recursive function, making an iteration.

Stack

  • It’s an array where data enters from an end and exits from the end.
    • The recently added data is removed first and the data stored earlier is taken out later, so it’s called Last In First Out, LIFO.
  • You can do DFS if you store visited vertices in the stack, and pop it out when you visit the next vertex.
    • If you fail to use stack and use queue instead, it ends up with becoming Breadth First Search (BFS).
      • Queue is First In First Out data structure.

Traverse

  • In Graph, DFS attempts to first reach the furthest vertex.
  • In Tree, it attempts to first go to the child node at the bottom.
  • This contrasts with Breadth First Search (BFS), because BFS visits the nearest neighbors first before going to the furthest places.
